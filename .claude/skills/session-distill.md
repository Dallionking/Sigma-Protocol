# Session Distill - Pattern Extraction from History

Analyze session transcripts to extract developer preferences and generate SLAS artifacts.

## Arguments

| Argument | Default | Description |
|----------|---------|-------------|
| `--sessions=N` | 50 | Number of recent sessions to analyze |
| `--bootstrap` | false | Initial analysis from JSONL transcripts |
| `--transcripts=PATH` | `~/.claude/projects/` | Path to transcript files |
| `--output=FORMAT` | yaml | Output format (yaml, json) |
| `--dry-run` | false | Preview without writing files |

## Execution Steps

### 1. Locate Session Transcripts

For `--bootstrap` mode, scan for JSONL transcript files:

```bash
# Find all Claude session transcripts
find ~/.claude/projects -name "*.jsonl" -type f 2>/dev/null | head -100

# Or use specific project path
find "$TRANSCRIPTS_PATH" -name "*.jsonl" -type f
```

### 2. Parse Transcripts

For each transcript file, extract:

#### Communication Patterns
- Average user message length (verbosity indicator)
- Use of punctuation and formatting
- Tone markers (direct vs. polite)

#### Autonomy Indicators
- Frequency of "just do it" vs. "ask first"
- Confirmation request patterns
- Correction frequency after autonomous actions

#### Quality Standards
- Code review comment patterns
- Error correction frequency
- Testing/verification requests

#### Frustration Triggers
- Negative feedback patterns
- Repeated corrections
- Explicit frustration expressions

#### Domain Expertise
- Topic frequency analysis
- Technical depth of questions
- Tool/framework preferences

### 3. Aggregate Patterns

Combine patterns across sessions with confidence weighting:

```yaml
patterns:
  autonomy_level:
    score: 0.85  # 0=ask everything, 1=fully autonomous
    confidence: 0.9
    evidence_count: 47

  communication:
    verbosity: minimal  # minimal|standard|detailed
    tone: direct  # direct|friendly|formal
    confidence: 0.85

  quality:
    testing_preference: high  # low|medium|high
    type_safety: strict  # loose|standard|strict
    documentation: minimal  # none|minimal|thorough

  frustrations:
    - trigger: "Repeated confirmation requests"
      frequency: 12
      severity: high
    - trigger: "Over-explanation of simple concepts"
      frequency: 8
      severity: medium

  domain_expertise:
    typescript: expert
    react: expert
    backend: intermediate
    devops: intermediate
```

### 4. Generate Developer Preferences Skill

Create `.claude/skills/developer-preferences/SKILL.md`:

```markdown
---
name: developer-preferences
description: Auto-generated developer preferences from session analysis
generated_at: {timestamp}
sessions_analyzed: {count}
confidence: {overall_confidence}
---

# Developer Preferences

> Auto-generated by SLAS from {count} sessions on {date}
> Confidence: {confidence}%

## Communication Style

{verbosity_rules}

## Autonomy Level

{autonomy_rules}

## Anti-Patterns to Avoid

{frustration_avoidance_rules}

## Quality Standards

{quality_rules}

## Domain Context

{domain_context}
```

### 5. Update Pattern Cache

Write to `docs/sessions/patterns/detected-patterns.json`:

```json
{
  "version": "1.0",
  "generated_at": "{timestamp}",
  "sessions_analyzed": {count},
  "patterns": {
    // ... aggregated patterns
  },
  "confidence_scores": {
    "overall": 0.85,
    "by_category": {
      "autonomy": 0.9,
      "communication": 0.85,
      "quality": 0.8
    }
  }
}
```

### 6. Update CLAUDE.md

Inject auto-generated section into CLAUDE.md:

```markdown
## Self-Learning Preferences (Auto-Generated)

> Last updated: {date} | Sessions analyzed: {count} | Confidence: {confidence}%

### Key Behaviors
- {behavior_1}
- {behavior_2}
- {behavior_3}

### Avoid
- {frustration_1}
- {frustration_2}

_Generated by SLAS. Run `/session-distill` to update._
```

## Bootstrap Mode

When running with `--bootstrap`:

1. Scan ALL available transcripts (not just recent)
2. Apply higher weight to recent sessions
3. Generate initial preference baseline
4. Create hook recommendations based on patterns

## Integration with SLAS

After distillation:
- Session start hook injects preferences
- Sigma-exit captures new patterns
- Periodic re-distillation updates preferences

## Privacy Note

- All analysis is local-only
- Transcript content is processed, not stored
- Only extracted patterns are persisted
- No data leaves the local machine
