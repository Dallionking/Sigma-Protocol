# Claude Code Integration Guide

This guide explains how Sigma Protocol integrates with Claude Code's native features for a seamless AI development experience.

## Overview

Sigma Protocol is designed to work natively with Claude Code's architecture, leveraging its:
- **Memory hierarchy** (CLAUDE.md, .claude/rules/)
- **Custom slash commands** (.claude/commands/)
- **Sub-agents and skills** (.claude/agents/, .claude/skills/)
- **Plugin system** for extended functionality

## Memory Hierarchy

Claude Code uses a layered memory system. Sigma Protocol aligns with this structure:

| Layer | Location | Purpose | Sigma Usage |
|-------|----------|---------|-------------|
| **Project Memory** | `CLAUDE.md` | Team-shared instructions | Generated by Step 12 |
| **Project Rules** | `.claude/rules/*.md` | Modular, topic-specific | Generated by Step 12 |
| **User Memory** | `~/.claude/CLAUDE.md` | Personal preferences | User-configured |
| **Local Memory** | `CLAUDE.local.md` | Private project prefs | User-configured |

### How Sigma Uses CLAUDE.md

When you run `@step-12-context-engine`, it generates a comprehensive `CLAUDE.md` that:

1. **Documents the project context** - Vision, tech stack, architecture
2. **Lists available commands** - All Sigma steps and utilities
3. **Provides workflow guidance** - Step ordering and dependencies
4. **Configures Ralph loop** - PRD-to-JSON conversion paths

Example generated CLAUDE.md structure:

```markdown
# Sigma Protocol - Claude Code Configuration

## Quick Start
- `/step-1-ideation` - Start product ideation
- `/status` - Check workflow progress

## Available Commands
[Full command table...]

## Workflow
[Step flow diagram...]

## Key Principles
[Value equation, HITL checkpoints, quality gates...]
```

### Modular Rules (.claude/rules/)

Sigma generates topic-specific rules in `.sigma/rules/` and mirrors them to `.claude/rules/` for Claude Code compatibility:

```
.claude/rules/
├── code-style.md        # Coding standards
├── testing.md           # Testing conventions
├── frontend.md          # Frontend-specific rules
├── backend.md           # Backend rules
└── security.md          # Security requirements
```

Each rule file supports **path-specific activation** via YAML frontmatter:

```markdown
---
paths:
  - "src/api/**/*.ts"
---

# API Development Rules
- All endpoints must include input validation
- Use standard error response format
```

## Custom Slash Commands

Sigma installs commands to `.claude/commands/` that work as native Claude Code slash commands.

### Command Structure

Each command is a Markdown file:

```markdown
---
description: "Run Sigma step-1 ideation"
allowed-tools:
  - Read
  - Write
  - Edit
  - Bash
---

# /step-1-ideation

[Full command content...]
```

### Using Commands

In Claude Code, simply type the command:

```bash
/step-1-ideation Build a task management app

/gap-analysis --spec=docs/prds/F01-auth.md

/sigma-continue --mode=auto
```

### Key Commands for Claude Code

| Command | Purpose |
|---------|---------|
| `/step-1-ideation` | Start product ideation |
| `/step-11-prd-generation` | Generate implementation PRDs |
| `/step-11a-prd-to-json` | Convert PRDs for Ralph loop |
| `/sigma-continue` | Resume unfinished work |
| `/sigma-simplify` | Clean up code (like code-simplifier) |
| `/gap-analysis` | Verify implementation completeness |

## Ralph Loop Integration

The Ralph loop enables autonomous implementation of PRD stories. Sigma provides full Ralph support for Claude Code.

### Setup

1. **Convert PRDs to JSON:**
   ```bash
   /step-5b-prd-to-json --all-prds   # For prototypes
   /step-11a-prd-to-json --all-prds # For implementation
   ```

2. **Run Ralph loop:**
   ```bash
   ./scripts/ralph/sigma-ralph.sh . docs/ralph/prd.json claude-code
   ```

### How Ralph Works with Claude Code

The `sigma-ralph.sh` script:
1. Reads `prd.json` for pending stories
2. Spawns a fresh Claude Code session per story
3. Provides context from `progress.txt` and `AGENTS.md`
4. Verifies completion via acceptance criteria
5. Updates story status and commits changes
6. Continues to next story

### Memory During Ralph Loop

- **Short-term:** `progress.txt` - Learnings between iterations
- **Long-term:** `AGENTS.md` - Patterns and conventions discovered
- **Work tracking:** `.sigma-manifest.json` - Active work state

## Sub-Agents

Sigma generates specialized sub-agents in `.claude/agents/`:

| Agent | Purpose |
|-------|---------|
| `sigma.md` | Main orchestrator |
| `sigma-executor.md` | Implementation specialist |
| `sigma-planner.md` | Strategic planning |
| `sigma-qa.md` | Verification and quality |

Use agents via Claude Code's `/agents` command or by referencing them in prompts.

## Plugin Compatibility

Sigma is compatible with Claude Code's plugin system. The `code-simplifier` plugin by Boris Cherny inspired our `@simplify` command.

To install official plugins:
```bash
claude plugin install code-simplifier
```

## Workflow Example

### Starting a New Project

```bash
# 1. Install Sigma Protocol
npx sigma-protocol

# 2. Initialize project (generates CLAUDE.md)
claude "Run /step-0-environment-setup"

# 3. Start ideation
claude "Run /step-1-ideation for [your product idea]"

# 4. Continue through steps
claude "Continue to step 2"
```

### Resuming Work

```bash
# Find where you left off
/sigma-continue

# Or auto-continue with Ralph
/sigma-continue --mode=auto --ralph
```

### Code Quality

```bash
# Simplify recent code changes
/sigma-simplify --scope=recent

# Verify implementation
/gap-analysis --spec=docs/prds/current.md
```

## Configuration

### Sigma Config (.sigma/config.json)

```json
{
  "platform": "claude-code",
  "ralph_mode": true,
  "auto_commit": true,
  "quality_threshold": 80
}
```

### Claude Settings

Configure Claude Code's behavior in `~/.claude/settings.json`:

```json
{
  "model": "claude-sonnet-4-5",
  "output_style": "explanatory"
}
```

### Auto-Handoff Configuration (Critical for Ralph Loop)

**Auto-handoff** enables automatic context management when Claude Code's context window fills up. This is **essential** for Ralph loop to handle large stories that exceed a single context window.

Add to `~/.claude/settings.json`:

```json
{
  "experimental.autoHandoff": {
    "context": 90
  }
}
```

**What this does:**
- When context reaches 90% capacity, Claude automatically hands off to a new session
- Preserves work in progress via git commits
- Maintains memory via `progress.txt` and `AGENTS.md`
- Allows large tasks to complete across multiple context windows

**Why it matters for Ralph:**
- Each Ralph iteration runs in a fresh context window
- Large user stories may require multiple sub-iterations
- Auto-handoff ensures no work is lost when context fills up
- Combined with git commits, provides checkpoint/resume capability

### Browser Integration

Claude Code supports multiple browser engines for UI verification:

| Engine | Setup | Best For |
|--------|-------|----------|
| **Playwright MCP** | Add Playwright MCP to settings | Full automation, headless testing |
| **Claude Browser** | Built-in | Quick visual inspection |

**Playwright MCP Setup:**

Add to your MCP configuration:

```json
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["@anthropic-ai/mcp-playwright"]
    }
  }
}
```

**Browser verification in Ralph acceptance criteria:**

```json
{
  "type": "ui-validation",
  "uiValidation": {
    "mode": "any",
    "route": "/dashboard",
    "checks": [
      { "type": "content-exists", "expectedText": "Dashboard" },
      { "type": "accessibility" }
    ]
  }
}
```

## Best Practices

### 1. Keep CLAUDE.md Updated

Run `@step-12-context-engine` after significant changes to regenerate context.

### 2. Use Plan Mode First

Always explore solutions before coding:
```
Shift+Tab → Plan Mode
"I want to build X, explore solutions"
```

### 3. Leverage Memory Files

- Document decisions in `CLAUDE.md`
- Log learnings in `progress.txt`
- Track patterns in `AGENTS.md`

### 4. Run Health Checks

```bash
npx sigma-protocol doctor
```

### 5. Keep Commands Focused

Each command handles one concern. Chain commands for complex workflows.

## Troubleshooting

### Commands Not Found

```bash
# Verify installation
npx sigma-protocol doctor

# Reinstall for Claude Code
npx sigma-protocol --target . --platform claude-code
```

### Memory Not Loading

1. Check `CLAUDE.md` exists in project root
2. Verify `.claude/` directory structure
3. Run `/memory` to inspect loaded files

### Ralph Loop Issues

1. Verify `prd.json` format matches schema
2. Check `progress.txt` for error messages
3. Ensure git is initialized
