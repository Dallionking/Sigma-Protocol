## Key Principles

### Value Equation (Hormozi)
Every feature must maximize:
```
Value = (Dream Outcome x Perceived Likelihood) / (Time Delay x Effort & Sacrifice)
```

### HITL Checkpoints
Commands pause for human approval at critical points. Never skip these.

### Quality Gates
Each step has verification criteria. Target: 80+/100 score.

### TODO Policy
- TODOs in production code must include an issue reference: `TODO(#issue-id): ...`
- Use GitHub issues for tracking (no unscoped TODOs).

## Execution Philosophy: Swarm-First

**Never work solo. Always use agent swarms for PRD execution.**

For all multi-agent work, use `Shift+Tab` to enter delegate mode. The orchestrator delegates to specialized agents -- never implements directly. See `.claude/agents/` for the full agent roster with tool restrictions, model assignments, and skill bindings.

Key rules:
- Every team includes Devil's Advocate + Gap Analyst as mandatory final gates
- Research-first planning: invoke `deep-research` before any planning session
- Match keywords to auto-invoke skills (see `.claude/rules/swarm-orchestration.md`)
- Assign skills by agent role (see `.claude/rules/swarm-orchestration.md`)
